# HelmGen

Веб-сервис для генерации, проверки и локального развёртывания Helm-чартов.

Сейчас в проекте уже работают:
- генерация Helm-chart;
- рекомендации по конфигурации;
- `helm lint`;
- `helm template`;
- `dry-run`;
- реальный `deploy`;
- `uninstall`;
- история chart;
- авторизация пользователей.

## Что есть в интерфейсе

- `Генератор`
  Здесь настраивается chart, выбираются тестовые сценарии, запускается генерация, проверка и скачивание архива.

- `Проверка и deploy`
  Здесь выполняются `helm template`, `dry-run`, `deploy` и `uninstall`.

- `История`
  Здесь видны ранее созданные chart и их статусы.

## Режимы запуска

Сейчас у проекта есть два нормальных режима.

### 1. Обычный режим

Подходит для:
- генерации;
- проверки;
- истории;
- интерфейсной работы без локального deploy через `minikube`.

Запуск:

```bash
docker compose up --build
```

После запуска открой:
- `http://localhost:3000` — интерфейс
- `http://localhost:8000/docs` — Swagger
- `http://localhost:8000/health` — healthcheck backend

В этом режиме всё работает в Docker:
- `frontend`
- `backend`
- `db`

### 2. Локальный deploy-режим

Подходит для проверки полного цикла:
- `render`
- `dry-run`
- `deploy`
- `uninstall`

Этот режим нужен, потому что локально `minikube` проще всего доступен для backend, запущенного на хосте.

Запуск:

```bash
make deploy-ready-local
```

Что делает эта команда:
- запускает `minikube`, если нужно;
- поднимает `db` в Docker;
- останавливает docker-контейнеры `frontend` и `backend`;
- запускает `frontend` локально;
- запускает `backend` локально.

Важно:
- этот режим альтернативный;
- не нужно параллельно запускать `docker compose up --build`;
- если нажать `Ctrl+C`, локальный deploy-режим остановится целиком.

Полезные команды:

```bash
make status-local-mode
```
Показывает, что сейчас живо в Docker.

```bash
make stop-local-mode
```
Останавливает `backend`, `frontend` и `db`, если они остались запущенными.

## Что делать после запуска

### Первый вход после введения авторизации

После открытия сайта сначала появится экран авторизации.

Есть два варианта:

1. `Регистрация`
  Создаёшь нового пользователя с email и паролем.

2. `Вход`
  Входишь в уже существующий аккаунт.

После успешного входа откроется основной интерфейс.

## Базовый сценарий использования

### Сценарий 1. Генерация chart

1. Открой вкладку `Генератор`.
2. Выбери один из тестовых сценариев или заполни форму вручную.
3. Нажми `Сгенерировать`.

Что должно произойти:
- chart сохранится в системе;
- справа обновится результат;
- в истории появится новая запись.

### Сценарий 2. Проверка chart

1. После генерации нажми `Проверить`.

Что должно произойти:
- запустится `helm lint`;
- справа появится результат проверки;
- статус chart обновится.

### Сценарий 3. Скачать архив

1. После успешной генерации или проверки нажми `Скачать`.

Что должно произойти:
- скачивается `.tgz` архив chart.

## Полный сценарий развёртывания

Этот сценарий лучше проверять в режиме:

```bash
make deploy-ready-local
```

Порядок:

1. На вкладке `Генератор`:
   - выбери deployable-сценарий, например `Публичный веб-сервис` или `Масштабируемый API`;
   - нажми `Сгенерировать`;
   - нажми `Проверить`.

2. Перейди в `Проверка и deploy`.

3. В блоке `Подключение к Kubernetes` нажми `Обновить`.

Что должно быть:
- `Helm: найден`
- `Kubeconfig: найден`
- `Context: minikube`
- `Кластер: доступен`

4. Нажми `Рендер`.

5. Нажми `Dry-run`.

6. Нажми `Развернуть`.

7. При необходимости нажми `Удалить release`.

## Как понять, что deploy реально сработал

Проверь через терминал:

```bash
helm list -n helmgen-demo
kubectl get all -n helmgen-demo
```

Если всё прошло успешно, release будет виден в `helm list`, а pod и service появятся в namespace.

## Если что-то не работает

### Не работает генерация или история

Скорее всего backend недоступен.

Проверь:

```bash
docker compose ps
```

Или, если ты в локальном deploy-режиме, проверь, что терминал с `make deploy-ready-local` всё ещё открыт.

### В `Проверка и deploy` кластер недоступен

Проверь:

```bash
minikube status
kubectl config current-context
kubectl cluster-info
```

И в UI нажми `Обновить` в блоке Kubernetes.

### Deploy падает с ошибкой

Сначала смотри прямо в UI:
- итог операции;
- блок `Последняя операция`;
- технический вывод в нужной вкладке.

Потом при необходимости проверь:

```bash
kubectl get all -n helmgen-demo
kubectl get events -n helmgen-demo --sort-by=.lastTimestamp
helm status orders-api -n helmgen-demo
```

### Нажал `Ctrl+C`, и непонятно, что осталось запущенным

Для локального deploy-режима:

```bash
make stop-local-mode
```

Для обычного docker-режима:

```bash
docker compose down
```

## Полезные команды

```bash
docker compose up --build
```
Обычный запуск проекта.

```bash
make deploy-ready-local
```
Локальный режим для проверки deploy через `minikube`.

make backend-test
```

make frontend-check

docker compose down

make stop-local-mode
```
